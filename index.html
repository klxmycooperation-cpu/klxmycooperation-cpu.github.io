SYSTEM 01 / CORPORATE INTEGRATION

«Честный знак»
как управляемый маршрут.

Сложный обмен данными, ЭЦП и проверки собраны в один управляемый маршрут.

USER → APP → SIGN → CHECK → DATA

  1. 01 / ACTION

    Точное действие

    Внутреннее приложениеВыберите операциюПроверить данныеПодписать запрос

  2. 02 / SIGN

    Подписанный запрос

    ЭЦП / готова→Сервисы маркировкиОперация передаётся только после подтверждения.

  3. 03 / GATE

    Проверка до записи

    ✓ обязательные поля✓ допустимый статус— повторы не найденыОшибка → безопасная остановка

  4. 04 / WRITEBACK

    Согласованный результат

    SQL Server / транзакция↙ ↓ ↘Delphi · ЭДО · внутренние системыНарушение условий → rollback.

Иллюстративная реконструкция, не production-скриншот.

  • SQL Server
  • Delphi
  • XML / ЭДО

OrderRadar

SOURCES → BASELINE → FILTER → TELEGRAM

  1. 01 / INPUT

    Разные публичные ленты

    Источник AИсточник BИсточник C↓Единый формат заказа

  2. 02 / MEMORY

    Новое, а не повтор

    baseline / сохранёнSQLite / seenNEW SIGNALСтарый поток и обычные дубли остаются за кадром.
